1. 概述與架構說明

1.1 API 基本資訊

項目說明
API 名稱stockin.data.create
功能說明工單完工入庫單資料新增
呼叫方向MES → ERP
通訊協定HTTP POST
Content-Typeapplication/json; charset=UTF-8
編碼格式UTF-8
同步/非同步同步呼叫 (Sync)

2. Request 規格

2.1 HTTP 請求

項目值
MethodPOST
Content-Typeapplication/json; charset=UTF-8

2.2 JSON 結構總覽

JSON Structure 
{
  "request": {
    "type": "sync",
    "key": "交易識別碼",
    "host": { // 來源系統資訊 },
    "service": { // 目標服務資訊 },
    "datakey": { // 資料索引鍵 },
    "payload": {
      "std_data": {
        "enterprise_no": "企業代碼",

    }
  }
}

3. Response 規格

3.1 JSON 結構

JSON Structure 
{
  "response": {
    "code": "回應代碼",
    "message": "回應訊息",
    "reqid": "請求識別碼",
    "srvver": "服務版本",
    "srvcode": "服務代碼",
    "payload": {
      "execution": {
        "status_code": "執行狀態碼",
        "sql_code": "SQL狀態碼",
        "description": "描述"
      },
      "datainfo": {
        "stock_in_no": "入庫單號"
      }
    }
  }
}

3.2 response 物件

欄位型別說明成功值
codestring回應代碼"019" (同步回覆)
messagestring回應訊息"Success" 或 "平台回覆同步結果"
reqidstring請求識別碼 (UUID)"c43bc4d3-2388-456a-8924-371c1ff3ab89"
srvverstring服務版本"1.0.0"
srvcodestring服務代碼"000" (成功)
payloadobject回應資料-

3.3 payload.execution 物件

欄位型別說明成功值
status_codestring執行狀態碼"0"
sql_codestringSQL 狀態碼"0"
descriptionstring錯誤描述 (成功時為空)""

3.4 payload.datainfo 物件

欄位型別說明範例值
stock_in_nostring入庫單號 (單別-單號)"5802-20260124066"
✅ 成功判斷條件
  • response.srvcode = "000"
  • response.payload.execution.status_code = "0"
  • response.payload.datainfo.stock_in_no 有值

4. 完整欄位說明

4.1 MES 欄位對應表

JSON 欄位MES 欄位說明
doc_type_noTRANSTYPE單據類別 (5802=完工入庫)
create_dateTRANSDATE單據日期
workstation_noOUTDEPID移出工作站
factory_noFACTORYID廠別
source_noTRANSTYPE + TRANSNOMES 來源單號
operator_noCREATER操作人員
wo_noMOTYPE + MONO工單號
item_noITEMID品號
stock_in_qtyTRANSQTY入庫數量
unit_noUNIT單位
warehouse_noINDEPID移入倉庫
location_noINSTORAGESPACE移入儲位
lot_noLOTNO批號

4.2 MES 回寫欄位

Response 欄位MES 欄位說明
stock_in_no (前段)SFT_TRANSORDER_LINE.TL011ERP 單據類別
stock_in_no (後段)SFT_TRANSORDER_LINE.TL012ERP 單據號碼
-SFT_TRANSORDER_LINE.TL013標記已送 ERP (設為 'Y')
⚠️ stock_in_no 解析規則

回傳的 stock_in_no 格式為 單別-單號，例如 5802-20260124066
MES 需拆解：
• TL011 = "5802" (單別)
• TL012 = "20260124066" (單號)

6. 錯誤碼與處理

6.1 srvcode 服務代碼

代碼說明
000成功
999一般錯誤
其他依 ERP 定義

6.2 execution.status_code

代碼說明
0執行成功
1執行失敗
其他依 ERP 定義

6.3 常見錯誤情境

description可能原因處理建議
工單不存在wo_no 格式錯誤或工單未建立檢查工單號碼格式
數量超過餘量累計入庫超過工單數量確認工單可入庫餘量
倉庫不存在warehouse_no 無效檢查倉庫設定
品號不存在item_no 無效檢查品號主檔
單位不符unit_no 與品號單位不符確認正確單位

7.2 Request 範例

{
  "request": {
    "type": "sync",
    "key": "取消交易識別碼",
    "host": { /* 同 stockin.data.create */ },

    },

  }
}
